# 写作风格指南 - 技术博客版

## ⚠️ 核心原则：去除"AI味"

### ❌ 禁止使用的内容

**1. 公众号营销风格**
- ❌ 煽情开头："你是否也有这样的困扰？"
- ❌ 虚假互动："欢迎在评论区分享"、"点个在看"、"转发给朋友"
- ❌ 营销话术："赋能"、"颠覆"、"极致"、"一站式"
- ❌ 排比句和空洞表述
- ❌ 过度使用 emoji 和感叹号

**2. AI 生成痕迹**
- ❌ "在当今快速发展的..."
- ❌ "随着...的不断发展..."
- ❌ "让我们一起探索..."
- ❌ "综上所述..."

**3. 虚假承诺**
- ❌ "效率提升 300%"
- ❌ "彻底改变你的工作方式"
- ❌ "从入门到精通"

### 3. 代码示例

**必须包含**：
- 完整可运行的代码
- 类型注解（如适用）
- 注释说明
- 错误处理

**示例**：
```python
def quick_sort(arr: list[int]) -> list[int]:
    """
    快速排序算法实现

    Args:
        arr: 待排序的整数列表

    Returns:
        排序后的列表
    """
    if len(arr) <= 1:
        return arr

    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]

    return quick_sort(left) + middle + quick_sort(right)
```

## 📋 文章结构规范

### 必需章节

1. **YAML 元数据**
2. **标题 + 封面图**
3. **核心要点** (Callout)
4. **技术说明**（方案架构/原理）
5. **安装配置**（分步骤，可复现）
6. **实战用法**（真实代码示例）
7. **常见问题** (FAQ)
8. **参考链接**（明链接）

### 可选章节

- 性能优化
- 高级用法
- 使用场景（适合/不适合）
- 对比分析

### 禁止章节

- ❌ "互动环节"
- ❌ "写在最后"
- ❌ "一句话总结"
- ❌ "下期预告"

## 🔗 链接规范

### ✅ 正确格式

**明链接**（推荐）：
```markdown
- **Ollama 官网**: https://ollama.com
- **GLM4 GitHub**: https://github.com/THUDM/GLM-4
```

### ❌ 错误格式

**Obsidian 内部链接**（不要用）：
```markdown
- [[Ollama 官方文档]]  ❌
- [[GLM4 使用指南]]    ❌
```

### 📌 参考链接章节

**只保留一个资源章节**，统一使用明链接：

```markdown
## 参考链接

- **资源名称1**: https://example.com
- **资源名称2**: https://example.com/docs
- **资源名称3**: https://github.com/example/repo
```

**删除这些冗余章节**：
- ❌ "相关资源" + "参考链接"（二选一）
- ❌ 使用 callout 包裹链接（直接列出）

**优先使用**：
- 自己的图床（file.costalong.com）
- GitHub CDN (cdn.jsdelivr.net)
- 其他稳定 CDN

### 2. 图片格式

```markdown

```

### 3. 图片位置

- **封面图**：标题后第一张
- **架构图**：技术说明章节
- **流程图**：安装配置章节
- **对比图**：性能/方案对比后
- **实战图**：实战用法章节

## 🎯 语言风格

### ✅ 推荐

- **直接陈述**："通过 Ollama 部署 GLM4 模型"
- **技术准确**："7B 参数，量化后 2.6GB"
- **可验证**："延迟 < 100ms（测试环境：M1 Max）"
- **实用主义**："适合处理公司内部代码"

### ❌ 避免

- **煽情**："你是否也遇到过..."
- **夸张**："效率提升 10 倍"
- **模糊**："大大提升了性能"
- **废话**："在实际应用中我们发现..."
