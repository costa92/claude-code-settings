# 7 维评分标准

每个维度 1-10 分，附具体扣分点和原文位置。

---

### 维度 1：可读性（Readability）

**评判要素**：

| 检查项 | 扣分标准 |
|--------|---------|
| 段落长度 | 超过 150 字的段落每个 -0.5 |
| 冗余表达 | 每处"其实就是""说白了就是""简单来说"等重复解释 -0.3 |
| 长句拆分 | 超过 60 字的单句每个 -0.5 |
| 节奏感 | 连续 3 段以上结构雷同（如全是"首先...其次...最后"）-1 |
| 过渡衔接 | 段落之间缺少承接，跳跃感明显，每处 -0.5 |
| 代码/正文比 | 代码块占比超过 50% 且缺少解释 -1 |

**评分锚点**：
- 9-10：一口气读完，没有卡壳的地方
- 7-8：整体流畅，个别段落可以精简
- 5-6：有些地方需要重读才能理解
- 3-4：频繁卡壳，信息密度不均
- 1-2：难以读完

### 维度 2：逻辑连贯性（Logic Flow）

**评判要素**：

| 检查项 | 扣分标准 |
|--------|---------|
| 论点支撑 | 核心论点缺少论据支撑 -2 |
| 论证链条 | 推理跳跃（A→C，缺少 B）每处 -1 |
| 信息完整 | 提到概念但未解释，读者可能不懂的，每处 -0.5 |
| 前后矛盾 | 前后表述不一致，每处 -2 |
| 结构呼应 | 开头承诺的内容在正文中未兑现 -2 |
| 章节顺序 | 章节排列不符合认知递进（如先讲高级用法再讲基础）-1 |

**评分锚点**：
- 9-10：论证严密，读完对主题有完整认知
- 7-8：整体连贯，个别地方可以补充
- 5-6：有断裂感，部分论点支撑不足
- 3-4：逻辑混乱，读完不知道重点
- 1-2：东拉西扯，无主线

### 维度 3：标题与 Hook（Title & Hook）

**评判要素**：

| 检查项 | 扣分标准 |
|--------|---------|
| 标题字数 | 不在 15-25 字范围 -1 |
| 关键词覆盖 | 标题缺少核心关键词 -1 |
| 利益承诺 | 标题没有明确的读者收益 -1 |
| 标题兑现 | 正文无法兑现标题承诺 -3 |
| Hook 吸引力 | 开头 100 字无法抓住注意力 -2 |
| Hook 关联度 | 开头与正文核心内容脱节 -1 |

**评分锚点**：
- 9-10：看到标题就想点，开头 3 秒就被抓住
- 7-8：标题有吸引力，开头能留住人
- 5-6：标题平平，开头没有特别抓人
- 3-4：标题模糊，开头冗长
- 1-2：标题与内容不匹配

### 维度 4：事实准确性（Factual Accuracy）

**评判要素**：

| 检查项 | 扣分标准 |
|--------|---------|
| 代码正确性 | 代码示例有语法错误或逻辑错误，每处 -2 |
| 命令准确性 | CLI 命令参数错误或已废弃，每处 -2 |
| 版本信息 | 提到的版本号/发布日期不准确，每处 -1 |
| 数据引用 | 引用的数据无来源或来源不可靠，每处 -1 |
| 链接有效性 | 外部链接失效（如有能力验证），每处 -0.5 |
| 概念准确 | 技术概念解释有误，每处 -2 |

**验证原则**：不确定的内容标 `⚠️ [需要验证]`，只对高风险内容（版本号、命令、重要数据）使用 WebSearch 验证，不逐条搜索。

**评分锚点**：
- 9-10：所有技术细节经得起推敲
- 7-8：整体准确，个别细节可以更精确
- 5-6：有几处需要修正的错误
- 3-4：多处事实错误，影响可信度
- 1-2：大量错误，不可发布

### 维度 5：AI 痕迹（AI Trace Detection）

**红旗词库**（出现即扣分，对照原文逐词核查）：

```
一级红旗（每个 -1.5）：
在当今...的时代 | 随着...的发展 | 让我们一起来看看 | 总而言之
综上所述 | 毋庸置疑 | 值得注意的是 | 不难发现
深度解析 | 全面梳理 | 一文看懂 | 手把手教你
万字长文 | 干货满满 | 建议收藏 | 这篇文章将

二级红旗（每个 -0.5）：
赋能 | 助力 | 一站式 | 全方位 | 深度赋能 | 无缝衔接
生态 | 闭环 | 抓手 | 底层逻辑 | 顶层设计 | 方法论
打通 | 沉淀 | 拉齐 | 对齐 | 颗粒度
范式 | 心智 | 链路 | 降本增效 | 快速落地

三级红旗（每个 -0.3）：
首先...其次...最后（连续使用）| 不仅...更是... | 一方面...另一方面...
事实上 | 实际上 | 需要指出的是 | 重要的是

四级红旗（每类 -0.2）：
"X 不仅是 Y，更是 Z" | "从 X 到 Y，我们可以看到 Z"
"在 X 领域，Y 已经成为 Z 的重要工具" | "接下来，让我们深入了解..."

新增：句子开头模式（每个 -0.5）：
当然 | 显然 | 不难看出 | 毫无疑问 | 众所周知

新增：数字列举偏好（每类 -0.5）：
有以下三点 | 主要包括以下几个方面 | 可以从以下几个角度
```

**结构性 AI 痕迹**：

| 检查项 | 扣分标准 |
|--------|---------|
| 段落结构雷同 | 连续 3+ 段"概念→解释→示例"结构 **-2**（原 -1）|
| 过度列举 | 每个论点都用 3 点或 5 点列举 **-2**（原 -1）|
| 语义重复 | 同一段内意思相近的句子出现 2 次以上 **-1**（新增）|
| 语气均匀 | 全文无情绪波动，像百科全书 -1 |
| 缺少个人视角 | 没有"我"的经验、观点或态度 -1 |
| 结尾套路 | "希望本文对你有帮助" 类结尾 -1 |
| 过渡词过多 | 几乎每段都以"此外""另外""同时"开头 **-1.5**（原 -0.5）|
| 标题层级机械 | 每个二级标题下必有三个三级标题 **-1**（原 -0.5）|

**最低分保护**：AI 痕迹维度最低扣到 1 分，不出现负分。

**快速模式**：输出"命中 X 个一级红旗，Y 个结构性痕迹，评分 Z/10"，不逐词列举。
**完整模式**：逐词列出命中词、所在段落位置和扣分值。

**评分锚点**：
- 9-10：像是真人写的经验分享，有温度，有个人立场
- 7-8：整体自然，少量痕迹可修
- 5-6：能感觉到 AI 味，但还能接受
- 3-4：明显 AI 生成，读者会有不适感
- 1-2：一眼 AI，不可发布

### 维度 6：平台适配（Platform Fit）

**目标平台**：默认微信公众号。用户可指定：`微信` / `小红书` / `知乎`。

---

**微信公众号标准**：

| 检查项 | 扣分标准 |
|--------|---------|
| 全文字数 | < 1500 字内容太薄 -1，> 5000 字过长 -1 |
| 外部链接 | 微信正文不支持外链，每处正文外链 -0.5（应改为搜索指引）；**参考资料区的 URL 不扣分、不删除**（Markdown 源文件必须保留完整 URL，外链转换由 wechat-article-converter 处理）|
| 图片密度 | 纯文字超过 800 字没有图/表/代码块 -1 |
| 代码块长度 | 单个代码块超过 30 行 -1（手机阅读体验差）|
| 标题层级 | 超过 3 级标题 -0.5（手机端层级过深难阅读）|
| 首图 | 缺少封面图建议 -0.5 |
| 摘要 | 缺少 120 字以内的摘要 -0.5 |
| 参考资料区结构 | 参考资料区出现图片 -1（应为纯文字列表）|

---

**小红书标准**：

| 检查项 | 扣分标准 |
|--------|---------|
| 字数 | > 1000 字过长 -1，< 200 字太浅 -1 |
| 封面图/首图 | 缺少封面图或首图 -2 |
| 话题标签 | 末尾缺少 3-5 个 # 话题标签 -1 |
| 分段密度 | 段落超过 3 行未换行 -0.5（小红书段落极短）|
| 表情符号 | 完全没有表情符号（小红书风格要求）-0.5 |
| 标题格式 | 标题没有数字或问句形式 -0.5 |
| 外链 | 出现外链 -1（小红书屏蔽外链）|

---

**知乎标准**：

| 检查项 | 扣分标准 |
|--------|---------|
| 字数 | < 500 字回答太浅 -1 |
| 开头格式 | 没有直接回答问题或给出结论 -1（知乎读者不耐烦）|
| 引用规范 | 有数据/观点未注明来源 -0.5 |
| 参考文献 | 缺少参考文献区 -0.5 |
| 学术语气 | 过于口语化，知乎受众偏好论述性语气 -0.5 |

**评分锚点**：
- 9-10：完全适配目标平台阅读体验
- 7-8：整体适配，个别地方需要调整
- 5-6：有几处不适配，需要修改
- 3-4：排版不适合目标平台
- 1-2：完全不适合目标平台

### 维度 7：原创性与差异化（Originality）

**评判核心**：这篇文章是否提供了读者在其他地方找不到的价值？

**评判要素**：

| 检查项 | 扣分标准 |
|--------|---------|
| 信息聚合 | 全文只是已有内容的搬运/整合，无独立分析 -3 |
| 独特视角 | 对已知信息缺少作者自己的判断或立场 -2 |
| 个人经验 | 没有一个真实案例、具体数字或亲身经历 -2 |
| 反直觉观点 | 全文结论都在意料之中，没有任何让人"哦原来如此"的内容 -1 |
| 类比与解释 | 没有用通俗类比解释复杂概念 -0.5 |
| 超越官方文档 | 内容基本等同于 README/官方文档的白话翻译 -2 |

**高分文章特征（8-10 分）**：
- 包含作者亲历的具体案例（有时间、地点、数字的真实事件）
- 对已知信息给出与主流观点不同的判断或结论
- 用通俗类比解释复杂概念，让读者"哦原来如此"
- 至少有一个让读者意外的发现或反直觉的洞察

**评分锚点**：
- 9-10：读完有"我从没这样想过"的感觉，值得分享
- 7-8：有自己的视角，信息有增量
- 5-6：信息量足够但没有差异化，可替代性高
- 3-4：基本是已有内容的重组，读者没有收获
- 1-2：完全是信息聚合，没有独立价值
