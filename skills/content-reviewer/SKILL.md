# Content Reviewer

**7 维评分审查文章质量，支持快速模式（默认）和完整模式，输出带前后对比的修改清单和发布建议**

---

## 使用方式

1. 用户提供文章（Markdown 文件路径 或 直接贴内容）及目标平台（默认微信公众号）
2. 检测调用模式（见下方）
3. 执行量化预检 → 7 维评分审查
4. 输出评分报告 + 修改清单 + 发布建议 + 联动出口

---

## 模式检测

```
用户调用 content-reviewer
        ↓
检测触发词：
  "快速" / "quick" / "速览" / "看一眼"  → 快速模式
  "完整" / "full" / "详细" / "深度"    → 完整模式
  未指定                               → 快速模式（默认）
        ↓
快速模式完成后，综合评分 < 55/70
        → 提示"未达发布线，输入「完整审查」后将自动修改"
  综合评分 < 35/70
        → 提示"分数过低，建议输入「完整审查」深度重写"
```

| 步骤 | 快速模式 | 完整模式 |
|------|---------|---------|
| 量化预检 | ✅ 执行 | ✅ 执行 |
| 通读全文 | ✅ 执行 | ✅ 执行 |
| 7 维评分 | ✅ 每维度只列 1 个最严重扣分点 | ✅ 所有扣分点 |
| 事实验证 | ❌ 跳过 | ✅ 高风险内容，附截图证明 |
| CLI 命令执行 | ❌ 跳过 | ✅ 本地→Docker 执行，截图 |
| 修改清单 | 仅🔴必须项，最多 5 条 | 🔴🟡🟢 全部 |
| AI 痕迹 | 统计命中数量 | 逐词列出（精确到段落）|
| 联动出口 | ✅ 显示 | ✅ 显示 |

## 审查流程

### 第零步：量化预检（评分前自动执行）

在进入主观评分之前，先用 Read 工具读取文章，统计以下客观指标：

| 指标 | 计算方式 | 参考值（微信） |
|------|---------|--------------|
| 全文字数 | 正文字数（不含代码块） | 1500-5000 |
| 代码块数量 | 统计 ``` 出现次数 / 2 | — |
| 最长代码块行数 | 逐个代码块计行 | ≤ 30 行 |
| 平均段落字数 | 正文字数 / 段落数 | ≤ 150 字 |
| 外部链接数量 | 统计正文中 http:// https:// 出现次数（参考资料区除外） | 0（微信正文不支持外链）|
| 标题层级深度 | 最深的 # 层级 | ≤ 3 级 |
| 图片数量 | 统计 ![]() 或 <img> | 每 800 字至少 1 张 |

将预检数据列在报告开头，后续评分直接引用，避免重复描述。

**文章类型自动识别**（影响部分维度的豁免规则）：

| 文章类型 | 识别特征 | 豁免规则 |
|---------|---------|---------|
| 教程/手把手 | 含「步骤」/「Step」/有序列表 | 结构雷同不扣分 |
| 观点/评论 | 第一人称多（≥5处）、无代码块 | 代码/正文比不计算 |
| 技术对比 | 含对比表格（≥2列×3行）| 过度列举不扣分 |

### 第一步：通读全文

- 记录第一遍阅读的直觉感受：流畅度、信息密度、是否想读下去
- 标记需要细查的段落位置

### 第二步：7 维逐项评分

每个维度 1-10 分，附具体扣分点和原文位置。

### 第三步：事实验证（仅完整模式，针对存疑内容）

见"事实验证与截图"节。

## 7 维评分标准

7 个维度（可读性、逻辑连贯性、标题与 Hook、事实准确性、AI 痕迹、平台适配、原创性与差异化），每维度 1-10 分，含扣分细则、评分锚点、AI 红旗词库（4 级）、三平台差异化标准（微信/小红书/知乎）等。

详见 [scoring-rubrics.md](references/scoring-rubrics.md)

## 评分阈值与建议

总分 70 分（7 维 × 10 分）。**发布线：55/70**

| 综合评分 | 建议 |
|---------|------|

| < 35 | ⛔ 建议重写 → **自动修改后重新评分** |

## 自动修改机制

**触发条件：综合评分 < 55/70**

评分完成后，不等待用户确认，立即执行以下流程：

```
综合评分 < 55/70
        ↓
第一轮自动修改：
  1. 按优先级顺序应用所有 🔴 必须修改项
  2. 同步应用所有 🟡 建议修改项
  3. 输出修改后的完整文章
        ↓
自动重新评分：
  对修改后的文章执行完整评分流程
        ↓
重新评分 ≥ 55/70？
  是 → 输出"修改完成，已达发布线（X/70）"+ 修改摘要
  否 → 执行第二轮修改（针对仍不达标的维度）→ 再次评分
        ↓
最多循环 3 轮，仍未达标则标注：
  "⚠️ 经 3 轮修改后仍为 X/70，以下维度建议人工介入：[列出维度]"
```

### 自动修改范围

| 优先级 | 修改内容 |
|--------|---------|
| 第一优先 | AI 痕迹：替换所有一级红旗词、修改结构性痕迹 |
| 第二优先 | 可读性：拆分超长段落（>150字）、拆分长句（>60字）|
| 第三优先 | 标题与 Hook：优化开头 100 字吸引力 |
| 第四优先 | 平台适配：将正文中的外链改为搜索指引（**不修改参考资料区的 URL**）、补充摘要 |
| 第五优先 | 逻辑连贯性：补充缺失的过渡句 |

**不自动修改的内容**（需要人工判断）：
- 事实准确性错误（涉及专业知识，不自动改）
- 原创性（需要作者补充个人经验，无法凭空生成）
- 标题兑现度（需要重写正文内容）

### 自动修改报告格式

修改完成后输出：

```markdown

**修改前评分**：X/70
**修改后评分**：Y/70（第 N 轮）

### 本轮修改内容

1. **[维度]：[问题描述]**
   - 修改前：`[原文]`
   - 修改后：`[改后文字]`

[... 所有修改项]

---

[修改后的完整文章内容]
```

## 联动出口规则

完整模式报告末尾的"下一步操作"根据评分动态生成：

| 条件 | 显示内容 |
|------|---------|
| 综合分 ≥ 55/70 | `转换微信格式` → 触发 wechat-article-converter |
| 综合分 ≥ 55/70 | `多平台分发` → 触发 content-repurposer |
| 标题分 < 7/10 | `优化标题` → 触发 wechat-seo-optimizer |
| AI 痕迹 < 6/10 | `去AI味重写` → 触发自动修改机制 |
| 综合分 < 55/70 | 自动触发"自动修改机制"，不等待用户确认 |

快速模式只显示"输入「完整审查」继续"，不展开联动出口。

## 注意事项

- 所有评分必须有具体依据，不打印象分
- 量化预检数据要准确计算，不要估算
- 修改建议必须给出"修改前→修改后"对比，必须可直接替换，不能只说"建议修改"
- 事实验证（完整模式）只针对高风险内容，不逐条搜索
- CLI 命令验证只执行只读命令，写操作跳过并标注
- AI 痕迹维度：只标记实际命中的红旗词，不要过度解读
- 原创性维度：客观评估信息增量，不因文章整体质量好而刻意给高分
- 平台适配维度：用户未指定平台时默认微信，指定后按对应标准
- 文章类型感知：识别到对应类型时应用豁免规则，并在报告中说明
- 审查完成后，综合分 ≥ 55 才展示联动出口；< 55 自动触发修改机制，不等用户确认
- 自动修改最多执行 3 轮，仍未达标则列出需要人工介入的维度
