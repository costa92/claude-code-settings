# Content Planner

**系统化管理公众号选题、排期和内容管线**

---

## 核心功能

### 1. 选题库构建

**输入**：用户的公众号定位、目标读者、已发布的热门文章（可选）

**选题来源矩阵**：

| 来源 | 方法 | 工具 |
|------|------|------|
| 热点追踪 | 监控 AI/技术领域热点事件 | WebSearch, ai-daily skill |
| 读者痛点 | 从评论区/社群提炼高频问题 | 用户提供 |
| 竞品分析 | 分析同类公众号的爆款选题 | WebFetch 抓取 |
| 个人经验 | 复盘自己的开发/使用经验 | 用户提供 |
| 系列规划 | 围绕一个大主题拆分系列文章 | 本 skill 生成 |
| 旧文翻新 | 基于新信息更新旧文章 | 用户提供旧文列表 |
| 技术文档 | 将官方文档转化为易读教程 | 用户提供文档链接 |
| 开源项目 | 分析热门项目/新发布的工具 | GitHub trending |

**选题评估四象限**：

```
                高传播性
                  │
    ┌─────────────┼─────────────┐
    │  🔥 爆款区   │  📢 话题区   │
    │  高实用+高传播│  低实用+高传播│
    │  → 优先做    │  → 蹭热点时做 │
    ├─────────────┼─────────────┤
    │  💎 口碑区   │  📦 填充区   │
    │  高实用+低传播│  低实用+低传播│
    │  → 稳定做    │  → 少做或不做 │
    └─────────────┼─────────────┘
                  │
                低传播性
        低实用性 ←───→ 高实用性
```

**选题卡片格式**：

```markdown
### 📌 选题：[标题想法]
- **类型**：教程 / 解析 / 观点 / 评测 / 资讯 / 盘点
- **象限**：🔥爆款 / 💎口碑 / 📢话题 / 📦填充
- **时效性**：长青 / 1个月 / 1周 / 即时
- **难度**：低(1天) / 中(2-3天) / 高(1周+)
- **目标读者**：[具体人群]
- **核心卖点**：[读者为什么要看这篇]
- **素材准备**：[需要准备什么]
- **关联文章**：[可以互相引流的历史文章]
- **优先级**：P0 / P1 / P2
```

**月度规划模板**：

```markdown

| W1 | MM/DD (周X) | 教程 | [选题名] | 📝规划中/✍️写作中/📋审核中/✅已发布 | |
| W1 | MM/DD (周X) | 观点 | [选题名] | | |
| W2 | | | | | |
| W3 | | | | | |
| W4 | | | | | |

## 内容配比（本月目标）
- 教程实战：X 篇 (XX%)
- 深度解析：X 篇 (XX%)
- 观点输出：X 篇 (XX%)
- 热点评论：X 篇 (XX%)
- 工具推荐：X 篇 (XX%)

## 系列文章进度
- [ ] [系列名] 第 X/Y 篇
```

**发布频率建议**：
- 个人号：每周 1-2 篇（保证质量）
- 团队号：每周 3-5 篇
- 日更号：每天 1 篇（需要储备池）

**最佳发布时间**：
| 时段 | 适合内容 | 原因 |
|------|---------|------|
| 周一-五 8:00-9:00 | 干货教程、工具推荐 | 通勤阅读，收藏备用 |
| 周一-五 12:00-13:00 | 轻度阅读、观点短文 | 午休碎片时间 |
| 周一-五 20:00-22:00 | 深度长文、系列文章 | 晚间沉浸阅读 |
| 周末 10:00-11:00 | 盘点、总结、故事型 | 周末休闲心态 |

### 3. 内容管线管理

**文章状态流转**：

```
💡选题 → 📋大纲 → ✍️初稿 → 📝审核 → 🎨排版 → 🖼️封面 → ✅发布 → 📊复盘
```

**管线看板（用 Markdown 表格模拟）**：

```markdown
## 📋 内容管线看板

### 💡 选题池 (Backlog)
| 选题 | 优先级 | 时效 |
|------|--------|------|
| [选题1] | P0 | 长青 |
| [选题2] | P1 | 1周 |

### ✍️ 写作中 (In Progress)
| 选题 | 预计完成 | 进度 |
|------|---------|------|
| [选题3] | MM/DD | 60% |

### 📝 待审核 (Review)
| 选题 | 文件路径 |
|------|---------|
| [选题4] | ./articles/xxx.md |

### ✅ 已发布 (Done)
| 选题 | 发布日期 | 阅读量 |
|------|---------|--------|
| [选题5] | MM/DD | - |
```

### 4. 系列文章规划

**系列化好处**：提高关注动机、减轻选题压力、建立专业认知

**系列规划模板**：

```markdown
## 📚 系列：[系列名称]

### 定位
- 目标读者：
- 系列价值：读完整个系列，读者能[获得什么]
- 预计篇数：X 篇
- 发布节奏：每周 X 篇

### 文章列表
1. **[标题]** — [核心内容] — 状态：✅/✍️/💡
2. **[标题]** — [核心内容] — 状态：
3. ...

### 系列间引流设计
- 每篇开头提示系列进度："本文是 [系列名] 第 X 篇"
- 每篇结尾预告下一篇
- 合集篇：系列完结后发一篇合集导航
```

## 工作流程

```
1. 了解用户的公众号定位和目标读者
   → 使用 AskUserQuestion 交互获取

2. 生成选题库
   → 结合热点（WebSearch）+ 用户输入 + 竞品分析
   → 输出选题卡片列表

3. 制定月度内容日历
   → 考虑时效性、内容配比、发布频率
   → 输出月度排期表

4. 维护内容管线
   → 保存到文件（如 content-calendar-2026-02.md）
   → 每次更新时读取并修改状态

5. 可选：配合其他 skill/agent
   → article-outline-agent: 展开具体选题
   → article-generator: 撰写文章
   → content-reviewer: 审核质量
   → wechat-seo-optimizer: 优化标题
   → md2wechat: 转换格式
   → content-repurposer: 多平台分发
```

## 关键原则

- **规划但不僵化**：日历是指南不是枷锁，热点来了要能灵活插入
- **储备先行**：保持至少 3-5 篇选题储备，避免"没东西写"的焦虑
- **数据驱动**：发布后复盘数据，用数据调整选题方向
- **系列思维**：单篇文章的价值有限，系列文章能建立认知和粘性
- **持续性 > 爆发性**：稳定更新比偶尔爆款更重要
