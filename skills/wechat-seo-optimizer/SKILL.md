# WeChat SEO Optimizer

**优化微信公众号文章的标题、摘要和关键词，提升打开率和搜索排名**

---

## 核心功能

### 1. 标题优化引擎

**输入**：文章内容（Markdown 文件）或文章主题描述

**分析维度**：

#### A. 标题公式库

| 公式 | 模板 | 示例 |
|------|------|------|
| 数字锚点 | X 个 [方法/技巧/工具] 让你 [获得好处] | 5 个 RAG 优化技巧让你的 AI 应用提速 3 倍 |
| 悬念缺口 | [已知现象]，但 [反常识结论] | 我用了 3 年 Kubernetes，最后回到了单体架构 |
| 痛点直击 | 还在 [错误做法]？[正确方案] 才是关键 | 还在手动部署？5 分钟搭建 CI/CD 才是正确姿势 |
| 身份锚定 | 作为 [身份]，我如何 [做到成就] | 作为独立开发者，我如何用 AI 每月节省 40 小时 |
| 对比张力 | [A] vs [B]：[结论/选择] | GPT-4o vs Claude 4：深度使用 30 天后的真实对比 |
| 时效热点 | [热点事件] + [你的解读角度] | OpenAI 发布 GPT-5：开发者应该关注这 3 个变化 |
| 教程承诺 | [从零/手把手] + [实现目标] + [可选：时间限定] | 从零搭建一个 AI Agent，只需要 100 行代码 |
| 颠覆认知 | [普遍认知] 是错的，[真相] 是... | "代码越少越好"是错的——论合理的代码冗余 |

#### B. 标题检查清单

```
✅ 必须满足：
- [ ] 字数 15-25（微信显示上限约 30 字，但 22 字左右效果最佳）
- [ ] 包含核心关键词（搜一搜排名）
- [ ] 有明确的读者利益承诺
- [ ] 正文能够兑现标题的承诺（不过度标题党）

⚠️ 加分项：
- [ ] 包含数字（3、5、7 比偶数更有吸引力）
- [ ] 有情绪词（真实、深度、避坑、实战、万字）
- [ ] 有身份代入（开发者、程序员、创业者）
- [ ] 有时间锚点（2026、最新、刚刚）

🚫 减分项：
- [ ] 模糊标题（"聊聊 AI 的一些事"）
- [ ] 过长标题（超过 28 字被截断）
- [ ] 纯英文标题（公众号中文读者占绝大多数）
- [ ] 使用已被用滥的词（赋能、干货、硬核、保姆级——视具体情况）
```

#### C. 输出格式

为每篇文章生成 **5 个标题方案**：

```markdown
## 标题优化建议

### 🏆 推荐标题
**[标题]**
- 公式：[使用的公式]
- 关键词：[覆盖的关键词]
- 预估效果：⭐⭐⭐⭐⭐
- 理由：[为什么推荐]

### 备选方案
1. **[标题]** — [公式] — ⭐⭐⭐⭐
2. **[标题]** — [公式] — ⭐⭐⭐⭐
3. **[标题]** — [公式] — ⭐⭐⭐
4. **[标题]** — [公式] — ⭐⭐⭐

### 避免使用
- ❌ **[反面示例]** — 原因：[为什么不好]
```

### 2. 摘要优化

**微信公众号摘要规则**：
- 显示在消息列表里，决定用户是否点开
- 最大 120 字，但前 40 字最关键（折叠后只显示一行）
- 不要重复标题内容
- 要补充标题没说的信息，形成"标题+摘要"的信息组合拳

**摘要公式**：

| 类型 | 策略 | 示例 |
|------|------|------|
| 利益补充 | 标题说了 what，摘要说 how | "附完整代码和部署指南，直接 copy 就能用" |
| 悬念延伸 | 标题制造好奇，摘要加强悬念 | "第 3 个原因是我没想到的，也是最关键的" |
| 社交证明 | 引用数据或权威 | "这套方案已在 3 个生产项目中验证，延迟降低 60%" |
| 共鸣引导 | 戳中目标读者 | "如果你也在小团队搞 AI 落地，这篇一定要看" |

**⚠️ 摘要相关性校验（必须执行）**：
生成摘要后，逐句检查以下规则：
1. **每个具体声称必须能在原文中找到依据**。禁止编造原文没有的数据或功能（如原文没提 QPS 就不能写"QPS 提升"）
2. **专业术语必须与原文一致**。不要用原文没出现的行业术语填充摘要
3. **利益承诺不能超出原文范围**。原文说"提升效率"，摘要不能升级为"彻底改变工作方式"
4. 如果不确定原文是否包含某个信息，**不写**比写错好

### 3. 关键词策略

**关键词分析流程**：

1. **提取文章核心关键词**（3-5 个）
2. **搜一搜热度估算**（基于通用搜索趋势推测）
3. **长尾关键词建议**（更具体的搜索词）
4. **关键词在文中的分布建议**：
   - 标题：必须包含 1 个核心关键词
   - 第一段：出现核心关键词
   - 小标题：分散包含长尾关键词
   - 自然密度：关键词占总字数 1-3%

**输出格式**：
```markdown
## 关键词策略

### 核心关键词
| 关键词 | 预估热度 | 竞争度 | 建议使用位置 |
|--------|---------|--------|-------------|

### 长尾关键词
| 关键词 | 建议使用位置 |
|--------|-------------|

### 当前文章关键词分布
- 标题包含：✅/❌ [关键词]
- 首段包含：✅/❌
- 小标题覆盖：X/Y 个
- 整体密度：X%（建议 1-3%）
```

### 4. 封面图建议

**微信公众号封面规格**：
- 头条封面：900×383 px（2.35:1）
- 次条封面：200×200 px（1:1）

**封面设计建议**：
- 技术文章：代码截图 + 大字标题 / 架构图 + 品牌色
- 观点文章：人物/场景 + 金句文字
- 教程文章：效果对比图 / Before-After
- 避免：纯文字封面、过于复杂的图片（手机缩略图看不清）

提供封面图的文字内容和布局建议，可配合 `nanobanana-skill` 或 `canvas-design` skill 实际生成。

## 完整优化报告格式

```markdown
# 📊 SEO 优化报告：[文章主题]

## 标题优化
[5 个标题方案 + 推荐]

## 摘要优化
- 推荐摘要：[文案]
- 备选摘要：[文案]

## 关键词策略
[关键词表 + 分布分析]

## 封面图建议
- 风格：[推荐风格]
- 文字内容：[封面上的文字]
- 布局：[简要描述]

## 发布时间建议
- 最佳时段：[工作日 8:00-9:00 / 12:00-13:00 / 21:00-22:00]
- 本文建议：[具体时间 + 原因]
```

## 关键原则

- **数据思维**：每个建议都要有理由，不能凭感觉
- **读者视角**：标题不是给作者看的，是给潜在读者看的
- **平衡艺术**：吸引力和诚实之间要平衡，不做标题党但也不做佛系标题
- **可验证**：建议提供 A/B 思路，让用户通过实际数据验证效果
