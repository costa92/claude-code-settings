# Content Repurposer

**将一篇文章转化为多个平台的内容格式，实现一鱼多吃**

---

## 使用方式

1. 用户提供源文章（Markdown 文件路径 或 直接贴内容）
2. 选择目标平台（可多选）
3. 生成各平台适配的内容

---

## 支持的目标平台

### 1. 小红书帖子

**格式要求**：
- 标题：带 emoji，20 字以内，吸引点击
- 正文：500-1000 字，口语化，分段短（2-3 行/段）
- 标签：10-15 个 # 话题标签，混合热门+长尾
- 图片建议：封面图 + 3-6 张信息图/步骤图
- 语气：亲切、实用、略带感叹，多用 emoji

**转化策略**：
- 提取文章中最实用的 3-5 个要点
- 去掉深度分析，保留"马上能用"的部分
- 加入个人体验/感受的口吻
- 结尾加互动引导："你们觉得呢？"/"收藏备用 📌"

**⚠️ 避免以下 AI 腔写法**（出现即重写）：
```
❌ "亲爱的开发者们" → ✅ 直接切入主题，不用称呼
❌ "分享一个改变我XX方式的宝藏工具" → ✅ 具体说是什么工具、解决什么问题
❌ "不仅...更是..." → ✅ 用短句直接描述
❌ "一旦用上，就回不了头" → ✅ 说具体哪个场景回不了头
❌ 通篇感叹号和"！！！" → ✅ 克制使用，1-2 个即可
❌ "码农们必看" → ✅ 用具体场景代替泛泛号召
```

**输出模板**：
```
📌 [标题 + emoji]

[开头 hook — 1-2 句，制造共鸣或好奇]

[要点 1 — emoji + 粗体标题]
[2-3 行说明]

[要点 2]
...

[总结/金句]

[互动引导]

#标签1 #标签2 #标签3 ...
```

### 2. Twitter/X Thread

**格式要求**：
- 第 1 条：hook + 承诺（"Thread 🧵"）
- 每条 280 字符以内（中文约 140 字）
- 总共 5-12 条推文
- 最后一条：总结 + CTA（关注/转发）
- 编号：1/N 格式

**转化策略**：
- 将文章核心论点拆解为独立的推文
- 每条推文自成一体，即使单独看也有价值
- 第 1 条决定生死——必须有强 hook
- 加入数据点、具体例子、反直觉观点
- 适度使用 emoji 做视觉分隔
- **hashtag 用英文**（中文 hashtag 在 Twitter 上无效），如 #AI #CodingTools

**⚠️ 避免以下 AI 腔写法**：
```
❌ "码农们必看！" → ✅ 用具体数据或结论做 hook
❌ 每条都以 emoji 开头 → ✅ 只在需要视觉分隔时用
❌ "🚀分享一个...的神器" → ✅ 直接说工具名和效果
❌ 最后一条 "觉得有用请 RT 🙏" → ✅ 提一个引发讨论的问题
```

**输出模板**：
```
1/ [强 Hook — 数据/问题/反直觉观点]

🧵 Thread: [主题]

2/ [第一个关键点]

3/ [展开/例子]

...

N/ 总结：
- [要点1]
- [要点2]
- [要点3]

觉得有用的话 RT 让更多人看到 🙏
```

### 3. 短视频脚本（抖音/B站/视频号）

**格式要求**：
- 时长：1-3 分钟（约 300-800 字口播稿）
- 开头 3 秒必须抓住注意力
- 结构：hook → 痛点 → 解决方案 → 行动号召
- 标注：画面提示、字幕重点、BGM 建议

**转化策略**：
- 只保留 1 个核心观点（短视频不适合多论点）
- 口语化改写，像在和朋友聊天
- 加入具体场景描述（"你是不是也遇到过..."）
- 节奏要快，每 15 秒一个信息点

**输出模板**：
```
## 短视频脚本：[标题]
**时长**: ~X 分钟 | **平台**: 抖音/B站/视频号

---

### 开头 (0-3秒) — Hook
🎬 画面：[描述]
🗣️ 口播：[台词]
📝 字幕重点：[关键词]

### 第一段 (3-30秒) — 痛点
🎬 画面：[描述]
🗣️ 口播：[台词]

### 第二段 (30秒-1分30秒) — 解决方案
🎬 画面：[描述 — 屏幕录制/演示/图解]
🗣️ 口播：[台词]

### 结尾 (最后15秒) — CTA
🎬 画面：[描述]
🗣️ 口播：[台词]
📝 字幕：[行动号召]

---
🎵 BGM 建议：[风格/节奏]
```

### 4. Newsletter 摘要

**格式要求**：
- 标题：简洁有力，包含关键词
- 摘要：150-300 字，概括核心价值
- 3 个关键要点 (bullet points)
- "为什么值得读"一段话
- 原文链接占位

**转化策略**：
- 提炼出"读者能从这篇文章获得什么"
- 用 BLUF (Bottom Line Up Front) 风格
- 要点要具体，不要泛泛而谈

### 5. 知乎回答

**格式要求**：
- 开头直接给结论（知乎读者讨厌绕弯子）
- 正文 800-2000 字，有深度
- 分层论证，有理有据
- 适当引用数据/来源增加可信度
- 结尾可以适度推广原文

**转化策略**：
- 设想一个对应的知乎问题
- 去掉公众号的营销感，增加学术/专业感
- 加入更多"为什么"的解释（知乎用户喜欢深度）

### 6. 微信朋友圈/短推文

**格式要求**：
- 200 字以内
- 1 个核心观点 + 1 个引发讨论的问题
- 可配 1-3 张图

## 工作流程

```
1. 读取源文章（Read 工具读取 Markdown 文件）
   → 提取核心论点、关键数据、最佳案例

2. 交互式选择目标平台（必须使用 AskUserQuestion）：
   问题："你想将这篇文章转化为哪些平台的内容？"
   选项（multiSelect: true）：
   - 小红书帖子
   - Twitter/X Thread
   - 短视频脚本（抖音/B站/视频号）
   - Newsletter 摘要
   - 知乎回答
   - 微信朋友圈

3. 为每个选中的平台生成适配内容
   → 严格遵守各平台的格式和反面示例规则

4. 用 Write 工具将每个平台的输出保存为独立文件：
   - {article-name}-xiaohongshu.md
   - {article-name}-twitter-thread.md
   - {article-name}-video-script.md
   - {article-name}-newsletter.md
   - {article-name}-zhihu.md
   - {article-name}-moments.md

5. 生成完成后，在回复中列出所有生成的文件路径
```

## 关键原则

- **每个平台有自己的语法**：不能简单缩写，要重新构思表达方式
- **核心保持一致**：所有平台传递同一个核心观点，但包装方式不同
- **先短后长**：从最短的格式（朋友圈）开始提炼，有助于找到核心
- **读者场景不同**：小红书是搜索驱动、Twitter 是信息流、视频号是推荐——开头策略完全不同
